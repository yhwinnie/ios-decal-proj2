    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-project-2-hangman-app" class="anchor" href="#project-2-hangman-app" aria-hidden="true"><span class="octicon octicon-link"></span></a>Project 2: Hangman App</h1>

<h2><a id="user-content-due" class="anchor" href="#due" aria-hidden="true"><span class="octicon octicon-link"></span></a>Due</h2>

<p>Monday, October 26 at 11:59 PM</p>

<h2><a id="user-content-description" class="anchor" href="#description" aria-hidden="true"><span class="octicon octicon-link"></span></a>Description</h2>

<p>In Homework 3, we gave you most of the basic logic for a functioning Hangman
game. As well, you built UI around the game. In Project 2, you'll be putting
everything together into a well-designed and fully-functional application.
Note that while many of the required features below have already been implemented
in Homework 3, some requirements have changed or been refined.</p>

<h2><a id="user-content-instructions" class="anchor" href="#instructions" aria-hidden="true"><span class="octicon octicon-link"></span></a>Instructions</h2>

<ol>
<li>Fork the assignment's repository from (<a href="https://github.com/iosdecal">https://github.com/iosdecal</a>).</li>
<li>Clone your forked repository to your local machine (the URL should contain
YOUR_USERNAME/ios-decal-..).</li>
<li><strong>Create Hangman app</strong>

<ul>
<li>Required

<ul>
<li>A "New Game" Button (to start a New Game)</li>
<li>A "Guess" Button (for the user to submit a Guess)</li>
<li>A method of guessing letters</li>
<li>A UILabel that indicates the incorrect guesses (on the same screen)</li>
<li>A square-dimensioned UIImageView that represents the "state" of the Hangman</li>
<li>Appropriate images for each "state" of the Hangman (we provide basic ones in /basic-hangman-img)</li>
<li>Use constraints - any and all views must scale to different devices</li>
<li>A working Hangman game</li>
<li>A win state, indicated by an Alert (Pop up box)

<ul>
<li>Should prevent additional guesses</li>
</ul></li>
<li>A fail state, indicated by an Alert (Pop up box)

<ul>
<li>Should prevent additional guesses</li>
</ul></li>
<li>A "Start Over" button</li>
</ul></li>
<li>Optional

<ul>
<li>A smart way for the user to guess letters (since a Textfield with keyboard entry is bad UX)

<ul>
<li>Warning: Having a button for each letter is the natural choice. However, consider the
difficulty of setting constraints on each button. Pursue at your own risk.</li>
</ul></li>
<li>Customized images for each "state" of the Hangman</li>
</ul></li>
</ul></li>
<li>Add, commit, and push your modified file(s) to your forked remote repository.</li>
</ol>
